# Clover

Clover is a writing support tool that combines NotebookLM, Claude Code, and TeX.

1. **Local TeX Environment**
   A TeX editor with PDF preview, running entirely on your machine. Work offline with full control over your files.

2. **NotebookLM-Style Source Management**
   Add experiment results and notes as sources, or use Deep Research to find relevant papers. Writing is grounded in your sources, preventing hallucination. New experiment data? Just add it to sources and instruct—the agent analyzes your data, generates figures, and updates your paper in one go.

3. **Claude Code Integration**
   Claude Code runs locally and reads your files directly. It handles the writing: fixing compile errors, deep research, brainstorming, and more. Write a dense paper from scratch in under an hour, or translate an entire paper effortlessly.

Available as desktop app (Electron) and web app (for remote servers).

## Clover vs Prism

Prism helps you write. Clover helps you turn data into papers.

|  | Clover | Prism |
|---|:---:|:---:|
| Write documents | ✓ | ✓ |
| Work with your local data | ✓ | - |

[Prism](https://openai.com/prism/) is OpenAI's cloud-based LaTeX editor—great for collaborative writing. But it can't access your local files.

Clover runs locally. Your experiment data, your notes, your CSVs—the agent reads them directly. Say "make a figure from this data and add it to section 3" and it's done.

## Features

- LaTeX editor with syntax highlighting (Monaco Editor)
- Real-time PDF preview
- AI agent integration (Claude Code)
- Deep research: Find academic papers and sources automatically
- Source management with graph-based tagging
- Project management with recent projects
- Dual deployment: Desktop (Electron) + Web (browser)

## AI Capabilities

The agent can help with:

- **Writing**: "Write the introduction", "Write a survey paper on coding agents"
- **Research**: "Deep research on transformers", "Find papers on X"
- **Editing**: "Translate to Japanese", "Make it more formal"
- **Debugging**: "Fix the compile error"

## Quick Start

### Requirements

- [Node.js](https://nodejs.org/en/download) 18+
- Your TeX environment ([TeX Live](https://www.tug.org/texlive/), MacTeX, MiKTeX, etc.)
- [Claude Code](https://claude.com/product/claude-code)

### Desktop App (Electron)

```bash
npm install
npm run dev
```

### Web App

```bash
npm install
npm run build:web
npm run start:server  # Starts on port 8080 (auto-opens browser)

# Options:
npm run start:server -- --port 3000    # Use specific port
npm run start:server -- --no-open      # Don't open browser
```

If port 8080 is in use, it automatically tries 8081, 8082, etc.

## Usage

### Creating a Project

1. Click "New Project"
2. Select a directory
3. Start writing LaTeX

### Keyboard Shortcuts

- `Cmd/Ctrl+S` - Save
- `Cmd/Ctrl+B` - Compile to PDF

### AI Agent

1. Click "Start" in the terminal panel
2. Enter instructions like "Write the introduction" or "Fix the compile error"
3. Auto-fix: Enable "Auto-start agent on compile error" in Settings

## Source Management

Sources (PDFs, papers, etc.) can be added via the Sources panel. Files are **not copied** - only the file path is stored in `.clover/sources.json`. The AI agent can read these sources when writing your paper.

Example `sources.json`:
```json
{
  "sources": [
    {
      "id": "uuid",
      "path": "/path/to/paper.pdf",
      "name": "paper.pdf",
      "addedAt": "2026-02-27T..."
    }
  ]
}
```

## Project Structure

```
project/
├── *.tex              # Your LaTeX files
├── CLAUDE.md          # Auto-generated by Clover
└── .clover/
    ├── config.json    # Project settings
    └── sources.json   # Research sources (file paths)
```
